# 01. 마크다운(markdown)
## 01.1 마크다운(markdown)이란?
**Markdown**은 텍스트 기반의 **마크업언어**로 2004년 존그루버에 의해 만들어졌으며 쉽게 쓰고 읽을 수 있으며 HTML로 변환이 가능하다. 
특수기호와 문자를 이용한 매우 간단한 구조의 문법을 사용하여 웹에서도 보다 빠르게 컨텐츠를 작성하고 보다 직관적으로 인식할 수 있다. 
마크다운이 최근 각광받기 시작한 이유는 깃허브(https://github.com) 덕분이다. 마크다운을 통해서 설치방법, 
소스코드 설명, 이슈 등을 간단하게 기록하고 가독성을 높일 수 있다는 강점이 부각되면서 점점 퍼져가게 된다.

## 01.2 마크다운(markdown)의 장점
- 별도의 도구없이 작성가능하다.
- 다양한 형태로 변환이 가능하다.
- 지원하는 프로그램과 플랫폼이 다양하다.
- 텍스트(Text)로 저장되기 때문에 용량이 적어 보관이 용이하다.
- 텍스트파일이기 때문에 버전관리시스템을 이용하여 변경이력을 관리할 수 있다.
- 
## 01.3 마크다운(markdown)의 단점
- 표준이 없기 때문에 도구에 따라서 변환방식이나 생성물이 다르다.
- 모든 HTML 마크업을 대신하지 못한다.

> ###### 제목 태그 6

### 3. 강조(Emphasis) 태그 
    이텔릭체 : *강조내용* 혹은 _강조내용_을 사용한다.
        굵게 :  **강조내용** 혹은 __강조내용__을 사용한다.
      취소선 : ~~취소내용~~을 사용한다.
        밑줄 : `<u>밑줄내용</u>`를 사용한다.
    
> ### 강조(Emphasis) 태그 적용 결과
> 이텔릭체 : *강조내용* 혹은' _강조내용_'을 사용한다.  
>    굵게 :  **강조내용** 혹은 '__강조내용__'을 사용한다.   
>  취소선 : ~~취소내용~~을 사용한다.  
>    밑줄 : `<u>밑줄내용</u>`를 사용한다.  

### 4. 목록(List) 태그
### - 순서가 있는 목록 (숫자)
    순서있는 목록은 숫자와 점을 사용한다. 숫자 순서는 내림차순으로 정렬된다.
    1. 첫번째
    3. 세번째
    2. 두번째

> ### 순서가 있는 목록 (숫자) 태그 적용 결과
> 1. 첫번째
> 3. 세번째
> 2. 두번째


### - 순서가 필요 없는 목록 (기호)
    기호를 혼합해서도 사용 가능하다.
    * 동그라미
       * 빈 동그라미
          * 네모

    + 동그라미
      + 빈 동그라미
        + 네모

    - 동그라미
      - 빈 동그라미
        - 네모

> ### 순서가 있는 목록 (숫자) 태그 적용 결과
>* 동그라미
>   * 빈 동그라미
>      * 네모
>+ 동그라미
>   + 빈 동그라미
>      + 네모
>- 동그라미
>   - 빈 동그라미
>      - 네모
